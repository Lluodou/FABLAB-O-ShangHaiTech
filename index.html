Lasercut

The goal of this assignment is to design and make some parts of splicing toy.

Design

The first step is to design it. First, we design a part that have many slots, which means this part is used to be a connector, connecting other parts together. First, we draw a large circle, then we draw a slot, then we copy it, and rotate it for 45°, then we choose two slots, and rotate it for 90°, and choose four slots and rotate it for 180°. Finally, we have 8 slots. After that, we can use tool in Fusion to make the edge of slots be round. And also, we can draw a litter circle in center to let this part have a small hole. Another part is a 90° corner. The way to design it is similar to design the first part. But one important thing is, must make sure the slots in both parts are identical (at least, slots should have same width).

From Fusion to CorelDraw

After finish design, we should save drafts in Fusion 360 as .dxf file. Right click on the draft and choose “save as .dxf”. Then we get .dxf file which can be used for laser cutting.

Final project

My final project is to make a rubber band cannon. This machine should be able to shoot rubber bands, and can be controlled by a joystick. I use 3 servo moter in the machine. So one of the main problem is to connect servo moter with the rest of the machine.

Design in hardware

First, create a board to be the gun barrel. Then create an entity to connect the board and contain a sevro moter.This sevro will be the trigger of the "cannon". And slot behind is used to connect another sevro moter, and this moter will control the elevation angle of the "cannon". Then create another entity to contain the sevro moter that control the elevation angle. Create another entity to connect the third sevro moter, this moter control the direction of the "cannon". Also, create another board to connect two entities. Create the last entity to contain the third moter. And another board to be the base of the last entity.

Design in software

The code of this machine is quite simple: There are 3 input: X axis, Y axis and K axis. X axis and Y axis are two analog input, which represent the skewing of the joystick. And K axis is an digital input, which represent the statement of the button (joystick itself is a button), the value of this input is 1 if we press the button, and the value of this input is 0 if we don't press the button. So we can let the value of X axis and Y axis input be the increment (or decrement) of the angle. What we need is to map the original input to increment of the angle. First, initialized the value and the statement of the software. Then read the input. And then deal with K axis signal. If fire_if=1, then trigger rotate, shoot the rubber band. Then change the elevation angle. The range of valid angle is 0 degree to 90 degree. When don't touch the joystick, the input of Y axis is 508~520, and the most common case is 514.
